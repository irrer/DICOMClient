Frequently Asked Questions


Will DICOM+ over-write my existing files?
In general, no - The program is designed to preserve existing files. Both anonymization and editing create new copies of the files. Before a new file is written, the program checks to make sure that the name it is using is unique.
The only exceptions are:

  • When editing a file, the user overrides the default and indicates that the original file should be over-written.
  • When the user repeatedly saves during an edit session, the same destination file is used.

Note that if the same series is anonymized multiple times, it will result in multiple sets of files. This is by design, as a common use case is to anonymize the same set of files multiple times with different user IDs to create test data.


How does DICOM+ anonymize UIDs?
It replaces all of them with new ones, maintaining the cross-references between files. This is especially important in radiotherapy, where a patient often has a set of files including images (CT, MR, etc.), RTSTRUCT, RTPLAN, RTDOSE, and other files that share UIDs between series.

To maintain the cross-references, all files for a given patient must be anonymized in the same session.

This is also required by the DICOM specification. See the DICOM standard:

  • Supplement 55: Attribute Level Confidentiality (including De-identification)
  • Supplement 142:Clinical Trial De-identification Profiles


How does DICOM+ construct file names?
File names are constructed so as to be human readable, and are formatted with the following fields (if they are present in the source DICOM file) separated by underscore:

  • Patient ID
  • Modality
  • Series Number
  • Instance Number
If the resulting file already exists, an integer is appended to the name to make it unique.


Does DICOM+ use 32 or 64 bit?
Either. The version of the Java runtime on your machine determines which will be used, but DICOM+ uses the same BAT and library files regardless. The advantage of 64 bit is the ability to work with larger data sets, but otherwised they behave identically.
